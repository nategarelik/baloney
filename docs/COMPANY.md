**Last updated:** 2026-03-20

## What Baloney Is

Baloney is an AI content detection platform. We tell people whether what they're looking at -- text, images, video -- was made by a human or generated by AI. We do this across three surfaces: a Chrome extension (inline detection on social media), a web app (baloney.app), and a mobile app (iOS + Android).

The cost of generating a lie is now zero. The cost of verifying truth shouldn't be.

### Detection Pipeline

Three-tier cascade ordered by reliability and cost:

```
Tier 1 -- Watermark Detection (deterministic, highest confidence)
  SynthID (Google Gemini watermarks, text + image)
  C2PA (Content Credentials, local byte parsing on mobile)

Tier 2 -- Commercial APIs (high accuracy, rate-limited)
  Pangram (text, 99.85% accuracy, 5 free/day)
  SightEngine (image + video, 98.3% ARIA benchmark)

Tier 3 -- Local Analysis (instant, unlimited, lower accuracy)
  Statistical 12-feature text analysis (burstiness, TTR, readability, etc.)
  FFT frequency analysis (spectral signatures in images)
  EXIF metadata forensics (camera provenance)
```

New detection methods slot into the cascade without disrupting existing ones. All thresholds and weights live in one file: `frontend/src/lib/detection-config.ts`.

## The Bigger Vision

### The Innovation We Discovered

While building Baloney, we accidentally built something more general: **a content processing runtime for the web.** The Chrome extension implements five primitive operations that compose into any content processing application:

| Primitive | What It Does |
|-----------|-------------|
| **OBSERVE** | Watch for content appearing on any webpage (MutationObserver, IntersectionObserver) |
| **EXTRACT** | Pull content out in processable form (base64, text, frames) |
| **ROUTE** | Send to any external processor (API dispatch via service worker) |
| **RENDER** | Overlay results back onto the page (dots, badges, toasts, blur) |
| **PERSIST** | Maintain state across sessions (chrome.storage, profiles) |

AI detection is one configuration of these primitives. Fact-checking, accessibility auditing, compliance monitoring, price intelligence, sentiment analysis -- all are different configurations of the same five operations.

### The Dual-Plane Architecture

Since Chrome 112 (April 2023), MV3 extensions run identically in headless mode. This means the extension runtime can operate at server-side fleet scale -- not just on one user's laptop. Combined with CDP (Chrome DevTools Protocol), this creates a dual control plane: CDP for orchestration, extension APIs for privileged observation.

No major product treats extension APIs as a co-equal control plane alongside CDP. Browserbase ($300M valuation) supports extensions as supplementary. Google's Project Mariner uses an extension and outperforms screenshot-based agents (83.5% vs 56% on WebVoyager). The extension trust boundary is an underexploited architectural advantage.

### Long-Term Platform Architecture

```
Layer 3: OPERATED INFRASTRUCTURE
         Observatory, web intelligence, compliance monitoring
         Enterprise products built on the platform

Layer 2: HOSTED PLATFORM
         Managed fleet of browser instances with extension runtime
         Session management, profile persistence, autoscaling

Layer 1: OPEN-SOURCE SDK
         Content Processing Runtime primitives
         Observe, Extract, Route, Render, Persist
```

Baloney is the first Layer 3 product. The platform enables everything else.

## Current Priorities

### Priority 1: Detection Accuracy
Get text and image accuracy above 90% on a 1,000+ sample benchmark. This is the foundation -- observatory data is only credible if the pipeline is trustworthy.

### Priority 2: User Growth
Grow organic scan volume across Chrome extension, web app, and mobile app. Every user scan feeds the observatory data. Distribution is the growth engine.

### Priority 3: Observatory Credibility
Make the aggregate data (Slop Index, trends, provenance) trustworthy and useful. Add statistical confidence indicators. Publish methodology. Target citation by researchers, journalists, or regulators.

### Priority 4: Platform Foundation (Future)
Extract the general-purpose primitives. Build fleet infrastructure. This activates when accuracy exceeds 90% and enterprise demand materializes.

## Regulatory Context

- **EU AI Act Article 50:** Enforceable August 2, 2026. Requires AI content marking. Fines up to 15M EUR / 3% global turnover.
- **California SB 942 / AB 853:** Effective August 2, 2026. AI transparency for image/video/audio.
- **C2PA metadata stripping:** 95%+ of platforms strip provenance data on upload. Third-party detection is the only verification fallback.
- **TikTok auto-detection rate:** 35-45% of AI content detected. 55-65% goes undetected. This gap is the market.
